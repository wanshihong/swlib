## 5. 后台管理系统

SWLib 的后台系统用于快速构建 CRUD 页面、运营后台和管理工具。它的重点不只是“把表生成出来”，而是让页面配置、字段展示、列表生命周期、动作按钮与模板扩展都具备明确边界。

### 快速入口

- 新建后台页面：看 `5.2` 与 `5.3`
- 字段选型与 VirtualField：看 `5.5` 到 `5.8`
- 动作按钮与保存钩子：看 `5.9` 与 `5.11`

### 常见误用

- 在列表格式化阶段逐行查库（N+1）
- 把虚拟值塞到真实字段，导致语义混乱

### 关联源码

- `Swlib/Admin/Controller/Abstract/AbstractAdmin.php`
- `Swlib/Admin/Config/PageConfig.php`
- `Swlib/Admin/Fields/*`

### 排障索引

- 列表性能差：优先检查 `VirtualField` 是否批量回填
- 字段不显示：检查字段显示控制与模板渲染条件

本文档覆盖：

- 后台页面生命周期
- 字段系统与选型
- `VirtualField` 的批量派生能力
- 模板、筛选、动作按钮与保存钩子
- 性能边界与完整案例

### 5.1 后台系统概述

#### 5.1.1 核心组件

| 组件                 | 作用        | 典型职责            |
|--------------------|-----------|-----------------|
| `AdminConfig`      | 后台站点级配置   | 标题、菜单、登录/退出路由   |
| `Dashboard`        | 后台首页控制器   | 首页与无权限页         |
| `Login`            | 后台登录控制器   | 登录、退出、修改密码      |
| `AbstractAdmin`    | 后台页面控制器基类 | 列表、表单、详情、生命周期组织 |
| `PageConfig`       | 页面配置对象    | 页面名称、表名、排序、分页   |
| `PageFieldsConfig` | 字段集合配置    | 管理字段列表与筛选集合     |
| `AbstractField`    | 字段基类      | 列表、表单、筛选、详情统一行为 |
| `ListRowManager`   | 列表行构建器    | 将 DTO 转换为可渲染字段行 |

#### 5.1.2 一个后台页面的组成

| 配置点                                       | 作用        | 何时使用       |
|-------------------------------------------|-----------|------------|
| `configPage()`                            | 页面基本信息    | 每个后台页都需要   |
| `configField()`                           | 字段配置      | 每个后台页都需要   |
| `join()`                                  | 列表页补充关联查询 | 需要关联表时     |
| `listsQuery()`                            | 列表查询附加条件  | 数据权限、默认筛选  |
| `detailQuery()`                           | 详情查询扩展    | 详情页需要额外数据时 |
| `insertBefore()` / `insertUpdateBefore()` | 保存前处理     | 校验、转换、补默认值 |
| `insertAfter()` / `insertUpdateAfter()`   | 保存后处理     | 缓存清理、联动任务  |

### 5.2 创建后台配置

后台站点必须有一个 `AdminConfig` 用于声明路由、菜单和站点级设置。

```php
<?php

namespace App\AdminDemo;

use Generate\RouterPath;
use Swlib\Admin\Config\AdminConfigAbstract;
use Swlib\Admin\Manager\AdminManager;
use Swlib\Admin\Menu\Menu;
use Swlib\Admin\Menu\MenuGroup;

class AdminConfig extends AdminConfigAbstract
{
    public function Init(AdminManager $layout): void
    {
        $layout->adminIndexUrl = RouterPath::AdminDemoDashboardIndex;
        $layout->logoutUrl = RouterPath::AdminDemoLoginLogout;
        $layout->loginUrl = RouterPath::AdminDemoLoginLogin;
        $layout->changePasswordUrl = RouterPath::AdminDemoLoginChangePassword;
        $layout->noAccessUrl = RouterPath::AdminDemoDashboardNoAccess;
    }

    public function configAdminTitle(): string
    {
        return '管理后台';
    }

    public function configMenus(): array
    {
        return [
            new MenuGroup(label: '用户管理', icon: 'bi bi-people')->setMenus(
                new Menu(label: '用户列表', url: RouterPath::AdminDemoUserUserAdminLists),
            ),
        ];
    }
}
```

#### 5.2.1 `AdminConfig` 配置项说明

| 方法 / 属性              | 作用                | 备注      |
|----------------------|-------------------|---------|
| `Init()`             | 配置后台入口、登录、退出等基础路由 | 必填      |
| `configAdminTitle()` | 后台标题              | 显示在页面顶部 |
| `configMenus()`      | 左侧菜单树             | 支持菜单分组  |

### 5.3 后台页面最小示例

```php
<?php

namespace App\AdminDemo\User;

use Generate\Tables\Main\UserTable;
use Swlib\Admin\Config\PageConfig;
use Swlib\Admin\Config\PageFieldsConfig;
use Swlib\Admin\Controller\Abstract\AbstractAdmin;
use Swlib\Admin\Fields\NumberField;
use Swlib\Admin\Fields\SelectField;
use Swlib\Admin\Fields\TextField;
use Swlib\Admin\Manager\OptionManager;
use Swlib\Table\Interface\TableInterface;

class UserAdmin extends AbstractAdmin
{
    protected function configPage(PageConfig $config): void
    {
        $config->pageName = '用户管理';
        $config->tableName = UserTable::class;
        $config->order = [UserTable::ID => 'desc'];
    }

    public function listsQuery(TableInterface $query): void
    {
        $query->addWhere(UserTable::STATUS, 1);
    }

    protected function configField(PageFieldsConfig $fields): void
    {
        $fields->setFields(
            new NumberField(UserTable::ID, 'ID')->hideOnForm(),
            new TextField(UserTable::USERNAME, '用户名'),
            new SelectField(UserTable::STATUS, '状态')->setOptions(
                new OptionManager(1, '启用'),
                new OptionManager(0, '禁用'),
            ),
        );
    }
}
```

### 5.4 列表页生命周期

后台列表页最容易被误用的地方是：开发者不知道“某段逻辑应该写在哪一步”。下面是推荐理解方式。

#### 5.4.1 生命周期步骤表

| 步骤 | 执行内容              | 可扩展点                                         | 注意事项                    |
|----|-------------------|----------------------------------------------|-------------------------|
| 1  | 初始化控制器            | `initController()`                           | 加载页面配置、字段配置、权限          |
| 2  | 收集列表字段            | `frameworkGetListsFields()`                  | 展示字段与基础查询字段分离           |
| 3  | 构建基础 query        | `field()`、`order()`                          | 查询字段来自字段配置              |
| 4  | 补充查询条件            | `join()`、`listsQuery()`                      | 只做基础 SQL 条件             |
| 5  | 接收筛选器             | `frameworkFilterRequest()`                   | 筛选属于 query 阶段           |
| 6  | 执行列表查询            | `selectAll()`                                | 得到当前页 DTO 列表            |
| 7  | 预处理关联字段           | `ControllerHelper::selectFieldBeforeQuery()` | 避免每行 `selectField()` 查询 |
| 8  | 处理 `VirtualField` | 内部虚拟字段解析流程                                   | 批量补齐派生列                 |
| 9  | 构建列表行             | `ListRowManager`                             | 此时字段值必须齐全               |
| 10 | 渲染 Twig           | 页面模板                                         | 模板只负责展示                 |

#### 5.4.2 常见需求写在哪

| 需求                    | 推荐写法                |
|-----------------------|---------------------|
| 列表默认 where 条件         | `listsQuery()`      |
| 关联表 join              | `join()`            |
| 单字段轻量格式化              | `setListFormat()`   |
| 跨表状态、聚合列              | `VirtualField`      |
| badge / 图片 / 自定义 HTML | `setTemplateList()` |

### 5.5 字段系统总览

#### 5.5.1 字段类型总表

| 字段类            | 列表 | 表单 | 筛选 | 详情 | 典型用途            |
|----------------|---:|---:|---:|---:|-----------------|
| `TextField`    |  是 |  是 |  是 |  是 | 普通文本            |
| `NumberField`  |  是 |  是 |  是 |  是 | 数字展示 / 输入       |
| `SelectField`  |  是 |  是 |  是 |  是 | 枚举、状态、关联文本      |
| `SwitchField`  |  是 |  是 |  是 |  是 | 开关状态            |
| `ImageField`   |  是 |  是 |  否 |  是 | 图片展示与上传         |
| `EditorField`  |  是 |  是 |  否 |  是 | 富文本内容           |
| `VirtualField` |  是 |  否 |  否 |  否 | 跨表状态、聚合列、批量派生字段 |

#### 5.5.2 字段选型表

| 需求       | 推荐方案                         | 备注                           |
|----------|------------------------------|------------------------------|
| 原表字段直接展示 | 普通字段类                        | 例如 `TextField`、`NumberField` |
| 状态枚举     | `SelectField`                | 静态枚举优先                       |
| 轻量格式化    | 普通字段 + `setListFormat()`     | 禁止查数据库                       |
| 关联表显示名称  | `SelectField::setRelation()` | 适合 ID -> 文本                  |
| 跨表状态、聚合列 | `VirtualField`               | 必须批量处理                       |

### 5.6 基础字段详解

#### 5.6.1 `TextField`

| 方法                  | 作用       | 默认值     | 备注                  |
|---------------------|----------|---------|---------------------|
| `setRequired()`     | 设置表单必填   | `false` | 作用于表单提交             |
| `setTemplateList()` | 自定义列表模板  | 文本模板    | 适合 badge、图片、复杂 HTML |
| `setListFormat()`   | 列表值轻量格式化 | `null`  | 不要在这里查数据库           |
| `hideOnForm()`      | 表单不显示    | `false` | 常见于 ID、系统字段         |
| `hideOnFilter()`    | 筛选不显示    | `false` | 减少无意义筛选项            |

#### 5.6.2 `NumberField`

| 方法                    | 作用     | 备注          |
|-----------------------|--------|-------------|
| `setMin()`            | 最小值    | 表单校验 + 前端属性 |
| `setMax()`            | 最大值    | 表单校验 + 前端属性 |
| `setDefault()`        | 默认值    | 创建表单默认填充    |
| `setListInlineEdit()` | 列表行内编辑 | 仅适合简单数字修改   |

#### 5.6.3 `SwitchField`

| 适合场景              | 不适合场景   |
|-------------------|---------|
| 布尔型启用/禁用、推荐/热门等状态 | 多个复杂状态值 |

#### 5.6.4 `EditorField`

| 适合场景            | 注意事项           |
|-----------------|----------------|
| 商品详情、文章正文、富文本说明 | 列表页通常隐藏，避免内容过长 |

#### 5.7.1 选型对比

| 固定枚举     | `setOptions()`  | 最简单、性能稳定             |

#### 5.7.2 常用方法

|-----------------|---------|-----------|

| `setAddQuery()` | 限制可选项   | 复杂后台表单    |

### 5.8 `VirtualField` 虚拟字段

`VirtualField` 用于“列表中要展示，但值不来自基础 SQL 查询”的列。典型场景包括：

- 当前期开期状态
- 聚合统计列
- 当前页批量派生摘要
- 依赖多表计算的 badge 文案

#### 5.8.1 普通字段与 `VirtualField` 对比

| 维度                  | 普通字段      | `VirtualField` |
|---------------------|-----------|----------------|
| 数据来源                | 基础 SQL 查询 | 查询后批量回填        |
| 是否参与基础 `field()` 收集 | 是         | 否              |
| 适合原表字段              | 是         | 否              |
| 适合跨表聚合展示            | 否         | 是              |
| 是否必须批量处理            | 否         | 是              |
| 是否推荐单行查库            | 否         | 明确不推荐          |

#### 5.8.2 默认配置

| 配置项        | 默认值                      | 说明              |
|------------|--------------------------|-----------------|
| 列表显示       | `true`                   | 默认用于列表          |
| 表单显示       | `false`                  | 第一阶段不参与表单       |
| 筛选显示       | `false`                  | 第一阶段不参与筛选       |
| 详情显示       | `false`                  | 第一阶段不参与详情       |
| 模板         | `fields/lists/text.twig` | 默认文本展示          |
| 默认值        | `''`                     | resolver 未命中时回退 |
| `batchKey` | 字段名本身                    | 高级模式可覆盖         |

#### 5.8.3 常用方法

| 方法                           |   是否必填 | 作用                |
|------------------------------|-------:|-------------------|
| `setBatchValueResolver()`    | 简单模式必填 | 直接返回 `[主键 => 值]`  |
| `setBatchKey()`              |      否 | 多个虚拟字段共享一批数据时用于分组 |
| `setBatchLoader()`           | 高级模式必填 | 一次性加载共享数据         |
| `setBatchRowValueResolver()` | 高级模式必填 | 从共享数据中提取当前字段值     |
| `setTemplateList()`          |      否 | 使用自定义列表模板         |
| `setDefault()`               |      否 | 设置未命中时默认值         |

#### 5.8.4 使用模式

##### 简单模式

```php
(new VirtualField('current_period_status', '当前开期状态'))
    ->setDefault('未开')
    ->setBatchValueResolver(function (array $rows): array {
        return [
            1001 => '待开奖',
            1002 => '到时间开奖【2026-03-11 12:30】',
        ];
    });
```

##### 分组模式

```php
$statusField = (new VirtualField('current_period_status', '当前开期状态'))
    ->setBatchKey('goods_period')
    ->setBatchLoader(function (array $rows): array {
        return $batchData;
    })
    ->setBatchRowValueResolver(function (array $row, array $batchData): string {
        return $batchData[$row['id']]['status'] ?? '';
    });
```

#### 5.8.5 适用场景选择

| 需求            | 推荐方式                                             |
|---------------|--------------------------------------------------|
| 单个虚拟列         | `setBatchValueResolver()`                        |
| 多个虚拟列共享一批查询结果 | `batchKey + batchLoader + batchRowValueResolver` |
| 原表字段轻量格式化     | 普通字段 + `setListFormat()`                         |

### 5.13 性能与最佳实践

#### 5.13.1 推荐做法

| 场景        | 推荐方式                        |
|-----------|-----------------------------|
| 列表跨表状态展示  | `VirtualField` 批量处理         |
| 多列依赖同一批数据 | 共享 `batchKey + batchLoader` |
| 展示复杂状态    | 在 resolver 中先映射，再交给模板展示     |
| 普通字段展示    | 使用现有字段类直接渲染                 |

#### 5.13.2 明确禁止

| 反模式                       | 原因        |
|---------------------------|-----------|
| 在 `setListFormat()` 中逐行查库 | 容易产生 N+1  |
| 在模板中计算聚合结果                | 展示层职责失衡   |
| 借用真实字段承载虚拟值               | 污染语义，难以维护 |
| 多个虚拟字段重复查询同一批数据           | 性能浪费      |

### 5.14 完整案例：商品当前开期状态

商品后台列表需要保留原有“状态 = 上架 / 下架”，同时新增“当前开期状态”列。该列数据来自 `goods_period`，属于典型的 `VirtualField` 场景。

#### 5.14.1 需求拆解

| 项目   | 内容                                       |
|------|------------------------------------------|
| 页面   | 商品管理列表                                   |
| 原字段  | 商品上下架状态                                  |
| 新字段  | 当前开期状态                                   |
| 数据来源 | `goods_period`                           |
| 实现方式 | `VirtualField` + 批量 resolver             |
| 模板   | `goods/goods_current_period_status.twig` |

#### 5.14.2 状态映射规则

| 条件                             | 展示文案                      |
|--------------------------------|---------------------------|
| 没有当前期                          | `未开`                      |
| `status = 1` 且 `open_mode = 2` | `到时间开奖【YYYY-MM-DD HH:mm】` |
| `status = 1` 且 `open_mode = 1` | `满额开奖【xx%】`               |
| `status = 2`                   | `待开奖`                     |

#### 5.14.3 示例代码

```php
(new VirtualField('current_period_status', '当前开期状态'))
    ->setDefault('未开')
    ->setTemplateList('goods/goods_current_period_status.twig')
    ->setBatchValueResolver(function (array $rows): array {
        $goodsIds = array_map(static fn(array $row): int => (int)($row['id'] ?? 0), $rows);
        $periodRows = $this->queryCurrentPeriodRows($goodsIds);

        $result = [];
        foreach ($goodsIds as $goodsId) {
            $result[$goodsId] = self::formatCurrentPeriodStatus($periodRows[$goodsId] ?? null);
        }

        return $result;
    });
```

### 5.15 FAQ

| 问题                               | 回答                                                  |
|----------------------------------|-----------------------------------------------------|
| 为什么不能在 `setListFormat()` 里查数据库？  | 因为它会对列表每行执行，容易形成 N+1                                |
| 为什么 `VirtualField` 不参与基础 SQL 查询？ | 因为它不是原表字段，而是查询后统一回填的展示列                             |
| 多个虚拟列如何共享一批查询？                   | 使用 `batchKey + batchLoader + batchRowValueResolver` |
| 模板能不能做状态计算？                      | 只允许轻量展示判断，复杂映射应在 resolver 中完成                       |
| 什么时候该重写 `lists()`？               | 绝大多数场景不需要。优先使用框架已有生命周期与 `VirtualField`              |
